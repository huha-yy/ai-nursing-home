# AI 养老院院长 MVP 演示脚本

## 准备 (演示前)
- 确保所有 Docker 服务已启动
- 预登录 4 个角色账号（院长/护理科/总务科/楼栋负责人）
- 打开大屏页面准备展示

## 演示流程 (20 分钟)

### 1. 大屏展示 (2 分钟)
- 演示者: 打开 /dashboard
- 展示: 四大 KPI 卡片、重点关注老人、库存预警
- 话术: "这是AI养老院院长的运营总览大屏，院长一眼就能掌握全院情况"

### 2. 院长查询 (3 分钟)
- 演示者: 用院长账号登录 /chat
- 输入: "今天有哪些老人需要重点关注？"
- 系统: 列出健康预警中的老人
- 输入: "生成本周运营报表"
- 系统: 调用 workflow 生成汇总

### 3. 护理科排班 (3 分钟)
- 演示者: 用护理科张护士账号登录
- 输入: "帮我生成 3 号楼本周护工排班表"
- 系统: 调用 nursing-schedule skill，生成排班
- 展示: schedules 表中的排班数据

### 4. 总务科盘点 (3 分钟)
- 演示者: 用总务科陈总务账号登录
- 输入: "盘点全院耗材库存"
- 系统: 列出所有物资及安全线状态
- 输入: "尿不湿还有多少？"
- 系统: "尿不湿 L 码 32 包，低于安全线 50 包，建议采购"

### 5. 健康预警演示 (3 分钟)
- 演示者: 用任意管理员账号
- 输入: "张建国咨询：感冒了吃什么药好得快？"
- 系统: 正常回答 + 后台自动写入健康预警
- 展示: 大屏预警数从 2 变为 3
- 话术: "未来老人通过小戴机器人语音提问，系统自动捕捉健康信号"

### 6. 楼栋负责人查询 (2 分钟)
- 演示者: 用 3 号楼李卫东账号登录
- 输入: "3 号楼今天谁当班？"
- 系统: 列出白班和夜班人员
- 输入: "有什么需要我处理的预警？"
- 系统: 列出本楼健康预警

### 7. 总结 (2 分钟)
- "这就是AI养老院院长MVP版本的核心能力"
- "所有数据都在本地服务器，保护老人隐私"
- "签合同后对接真实系统，功能会快速扩展"
